# 02-mistakes-and-fixes

Purpose:
Document mistakes, bugs, and how they were fixed so they are not repeated.

Examples of notes:
- Misconfigured optimizer leading to training instability and how it was fixed.
- Data preprocessing bug that caused label leakage and the correction steps.
- Wrong assumption about an API that broke production and the proper usage.

Structure:
YYYY-MM-DD-title.md

Example:
- 2026-01-22-bad-dataloader-sampler-in-fl.md
